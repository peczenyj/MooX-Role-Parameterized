# Minimum Perl 5.12 + Release Automation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Raise the `MooX-Role-Parameterized` Perl floor to 5.12 and add a tag-triggered workflow that releases to CPAN and creates a GitHub release.

**Architecture:** Two sequential CPAN releases. `v0.502` is a rehearsal: it adds the release automation, `xt/` author tests, and develop-phase metadata, and fixes the `$VERSION` typo — without changing the supported Perl version. `v0.600` then raises the floor to 5.12. This is infrastructure work: it is verified by running the existing test suite and the new author tests, not by red-green TDD. No task in this plan tags or pushes a release — tagging is a manual maintainer action gated behind the two checkpoints.

**Tech Stack:** Perl, `ExtUtils::MakeMaker`, GitHub Actions, `Test::Perl::Critic`, `Test::PerlTidy`, `CPAN::Uploader`.

**Source spec:** `docs/superpowers/specs/2026-05-15-min-perl-5.12-design.md`

## Task 5: Add the release workflow

**Files:**
- Create: `.github/workflows/release.yml`

- [ ] **Step 1: Create `.github/workflows/release.yml`**

Create the file with exactly this content:

```yaml
name: release

on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  release:
    runs-on: ubuntu-latest
    container:
      image: perl:latest
    steps:
      - uses: actions/checkout@v4

      - name: Configure git
        run: git config --global --add safe.directory '*'

      - name: perl -V
        run: perl -V

      - name: Install dependencies
        run: |
          cpanm -nq --with-develop --installdeps .
          cpanm -nq CPAN::Uploader

      - name: Verify tag matches $VERSION
        run: |
          tag="${GITHUB_REF_NAME#v}"
          for mod in \
            lib/MooX/Role/Parameterized.pm \
            lib/MooX/Role/Parameterized/Mop.pm \
            lib/MooX/Role/Parameterized/With.pm
          do
            ver=$(perl -ne 'if (/\$VERSION\s*=\s*"([^"]+)"/) { print $1; exit }' "$mod")
            if [ "$ver" != "$tag" ]; then
              echo "::error::Version mismatch in $mod: \$VERSION='$ver' but tag='$tag'"
              exit 1
            fi
          done
          echo "All three modules are at version $tag"

      - name: Run test suite
        run: prove -lr t

      - name: Run author tests
        run: prove -l xt

      - name: Build the distribution tarball
        run: |
          perl Makefile.PL
          make manifest
          make dist

      - name: Upload to CPAN
        env:
          PAUSE_USERNAME: ${{ secrets.PAUSE_USERNAME }}
          PAUSE_PASSWORD: ${{ secrets.PAUSE_PASSWORD }}
        run: |
          cpan-upload --user "$PAUSE_USERNAME" --password "$PAUSE_PASSWORD" MooX-Role-Parameterized-*.tar.gz

      - name: Create GitHub release
        uses: softprops/action-gh-release@v2
        with:
          files: MooX-Role-Parameterized-*.tar.gz
          generate_release_notes: true
```

`permissions: contents: write` is required for `action-gh-release` to create the release with the default `GITHUB_TOKEN`.

- [ ] **Step 2: Verify the YAML is well-formed**

Run:
```bash

```
Expected: `YAML OK`. This workflow cannot run locally — it is exercised end-to-end when `v0.502` is tagged (Checkpoint 1).
